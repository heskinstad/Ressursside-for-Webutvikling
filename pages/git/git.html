Git/GitHub

02/10/2018

Git

Git er en programvare som brukes for å kunne gjøre versjonskontroll av datafiler, eksempelvis i et prosjekt, noe som kan være en absolutt nødvendighet (spesielt i prosjekter der flere utviklere samarbeider). Git ble opprinnelig påbegynt av mannen bak Linux, Linus Torvalds. Det at Git gjør versjonskontroll av filer, betyr at den som en del av backup-prosessen lagrer alle forandringer, slik at det på den måten skal mye mer til for å "miste" data eller ikke kunne revertere endringer. Forøvrig er Git også gratis og dens kildekode åpen. Gits offisielle nettsted beskriver seg selv slik: "Git is a free and open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency."

Noe jeg synes er en stor fordel med Git, er nettopp de store mulighetene har man for å samarbeide med andre, med tanke på versjonskontrollen og selve lagringsporsessen. Git er nemlig bygd opp slik at flere utviklere deler samme "repository", der de kan gjøre endringer samtidig. Når endringer blir gjort, får filene IDer som skiller dem fra hverandre, også om det bare blir gjort endringer i filene. På den måten har alle som samarbeider tilgang til filer "bakover i tid". Samtidig sørger Git for at man aldri overskriver hverandres kode ved et uhell. Har man en gammel versjon av filen/prosjektet gir Git beskjed om dette samt at det er mulig å merge de forskjellige versjonene sammen.

Endringer i Git-filer skjer ved hjelp av øyeblikssbilder (snapshots) av dataen som er blitt forandret. Ifølge Git-teamet selv er "Git-metoden" (en "strøm" av øyeblikksbilder) overlegen mer tradisjonell "deltabasert versjonskontroll" (at informasjonen blir lagret ved å spore alle endringer gjort på hver fil over tid). Arbeidsflyten ved bruk av Git fungerer mer eller mindre slik:

  • Modifisering/endringer av filer gjøres (lokalt).
  • De områdene man vil oppdatere av filen(e) velges. Ikke noe annet blir oppdatert i neste fase.
  • Et øyeblikksbilde av det valgte området fra forrige fase blir tatt, for deretter å laste opp dette til ens Git-oppbevaringssted. Det er etter dette stadiet at andre kan se forandringene (din siste "commit").

GitHub

Som navnet tilsier, så er GitHub en nettbasert vertstjeneste for Git, og er på dette tidspunktet (2018) den største verten for kildekode i verden. GitHub tilbyr alt av Gits standardfunksjonalitet og i tillegg noen ekstra funksjoner som forenkler samarbeidprosessen og delingsmulighetene.

GitHub fungerer ved at man først lager en "repository". Dette er selve prosjektet man skal arbeide med. Ved å ha prosjektet åpent er det mulig for andre å ikke bare ha tilgang til koden, men også å kunne komme med forslag til endringer (men den som har laget prosjektet må fortsatt "godkjenne" dette for at selve repository-koden skal bli endret). Prosjektet/repositorien kan deles opp i ulike grener ("branches"). Slike grener inneholder en kopi av originalfilene, men forandringer gjort på disse grenene påvirker ikke hovedgrenen (master branch). På denne måten kan man derfor gjøre alt mulig av forandringer på prosjektgrenen sin uten å risikere at filene uheldigvis blir merget sammen med hovedfilene i prosjektet. GitHub har også mulighet for å vise alle forskjeller mellom en sidegren og master branch før de merges sammen.
